Discuss the scope of var, let, and const

1. Discuss the scope of var

Var is a functional scope

If Var create a vaiable, The variable can be reassigned

Var can declared without initialization

It can be accessed without initialization as its default value is “undefined”.

Var declarations are globally scoped or function/locally scoped.

2. Discuss the scope of let

Var is a block scope

If Var create a vaiable, The variable can be reassigned

Var can declared without initialization

It can be accessed without initialization otherwise it will give "referenceError"..

let is now preferred for variable declaration.

3. Discuss the scope of const

Var is a block scope

If Var create a vaiable, The variable cannot be reassigned another value

Const declarations can only be accessed within the block they were declared.

It cannot be accessed without initialization, as it cannot be declared without initialization.

const declarations share some similarities with let declarations.

Tell us the use cases of null and undefined

1. Discuss the use cases of null

Null is an object in JavaScript and represents primitive data types

Null indicates the intentional absence of a value

When used in conditional logic null will return false

Null are falsy by default

Null will be converted to 0 behind the screens.

2. Discuss the use cases of undefined

Undefined is a primitive value automatically assigned by JavaScript to indicate the unintentional absence of any object value.

When used in conditional logic undefined will return false

Undefined are falsy by default

Undefined will result in NaN (not a number)

If not assigned a value it is by default given a value of undefined

What do you mean by REST API?

1. Discuss About REST API?

API means Application Programming Interface

It is a series of rules that guides communication between applications and devices

The API takes a request from a user, communicates with the server, and then delivers the response to that request.

A REST API is an API that features representational state transfer design.

REST APIs take in requests for a resource (any type of data or information) and then send back all relevant information
